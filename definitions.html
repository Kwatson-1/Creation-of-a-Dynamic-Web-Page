Programming concept definitions

Authentication: is a means by which a server will attempt to identify who is accessing their information or site. It is also used by users to determine whether a server is the system it claims to be.

Web security: refers to the protection of users, computer systems and networks from damage to or theft of software, hardware, or data.

HTTP: is the set of rules for transferring files like text, images, sound, video, and other multimedia files over the web. HTTP follows a client-server model where a client opens a connection to make a request and will then wait until it receives a response.

Session management: is the process by which multiple requests to a web-based application or service from a single user or entity are securely handled. Websites use HTTP to communicate and a session is series of HTTP requests and transactions initiated by the same user.

Stateless programming: refers to a program or application that ‘does not maintain a state‘ or when the infrastructure of a system prevents a program from maintaining state. This means that data is not stored or carried from one session to the next.

Analysis places emphasis on investigating a problem and its requirements for the purpose of gathering information to inform a conclusion or recommend a specific course of action.

Design places emphasis on the conceptualization of a solution to meet requirements and often does not concern itself with implementation.
